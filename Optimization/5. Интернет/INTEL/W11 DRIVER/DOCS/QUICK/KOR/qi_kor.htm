인텔® 네트워크 어댑터 설치

이 페이지에서는 인텔® 이더넷 장치 및 소프트웨어에 대한 간단한 설치 지침을 제공합니다.

자세한 설치 및 구성 정보는 다음을 참조하십시오:

  • 인텔® 이더넷 어댑터 및 장치 사용자 가이드 – 인텔 리소스 및 문서 센터에서 제공
  • Linux* 및 FreeBSD* 드라이버 tarball 내의 README 파일들

인텔® 이더넷 어댑터 설치

어댑터 설치

  1. 컴퓨터를 끄고 전원 코드를 뽑습니다. "핫 플러그"를 설치하려면 먼저 서버 설명서를 참조하십시오.
  2. 컴퓨터 덮개를 제거한 다음 해당 어댑터와 일치하는 슬롯에서 어댑터 슬롯 덮개를 제거합니다. 어댑터의 커넥터보다 더 긴 슬롯을 사용하는 것은 문제가 없지만, 커넥터보다 짧은 슬롯에는 설치하지 마십시오.
  3. 어댑터를 슬롯에 잘 고정시킨 다음 브래킷을 단단히 끼웁니다.
  4. 컴퓨터 덮개를 다시 덮고 전원 코드를 연결합니다.

메자닌 카드 설치

  1. 컴퓨터를 끄고 전원 코드를 뽑습니다.
  2. 컴퓨터 덮개를 제거합니다.
  3. 메자닌 보드의 PCIe 커넥터를 마더보드의 커넥터 및 플라스틱 스탠드오프 4개에 맞춥니다.
  4. MB 커넥터에 완전히 삽입될 때까지 어댑터를 누르고 4개의 스탠드오프에 끼웁니다.
  5. 컴퓨터 덮개를 다시 덮고 전원 코드를 연결합니다.

네트워크 케이블 연결

구리 어댑터:

10 Mbps 네트워크의 경우 카테고리 3, 4 또는 5 꼬인 쌍 이더넷(TPE) 네트워크 케이블을 사용합니다.
100 또는 1000 Mbps 네트워크와 주거 지역 네트워크에는 카테고리 5(4 페어) 케이블을 사용합니다.
10 Gbps 네트워크의 경우에는 카테고리 6, 카테고리 6a 또는 카테고리 7 이더넷 케이블을 사용합니다.

SC 커넥터가 있는 광섬유 어댑터:

광섬유 커넥터 덮개를 분리하여 보관하십시오. 1000Base-SX 듀플렉스 유형 SC 광섬유 커넥터를 어댑터 브래킷의 TX/RX 포트에 끼우십시오.

LC 커넥터가 있는 광섬유 어댑터:

광섬유 커넥터 덮개를 분리하여 보관하십시오. 어댑터에 맞는 케이블로 LC 커넥터를 어댑터에 삽입합니다. 길이 제한을 포함한 어댑터의 광학 사양이 케이블 연결 사양과 맞는 경우에는 SC와 같이 다른 커넥터 종류에 사용하는 전환 케이블을 사용할 수도 있습니다.

SFP+ 플러그 가능 광섬유:

모듈 제조업체의 지침에 따라 모듈을 설치하고 케이블을 연결합니다.

CX4 커넥터:

물리적인 케이블이 IEEE Std 802.3ak, Clause 54 사양을 충족해야 합니다. 필요한 케이블 커넥터는 IEC 61076-3-113으로 규정된 MDI 래치형 소켓입니다.

컴퓨터를 시작하고 아래의 드라이버 설치 안내를 따르십시오.

드라이버 설치

Microsoft* Windows* 운영 체제

Microsoft Windows용 소프트웨어는 다음과 같은 패키지로 제공됩니다.

  • Windows 드라이버만 제공. 이 옵션에는 인텔® PROSet 소프트웨어가 포함되지 않습니다.
  • 인텔 PROSet만 제공. 이 옵션에는 Windows 기본 드라이버가 포함되지 않습니다. 인텔 PROSet을 설치하기 전에 기본 드라이버를 설치해야 합니다.
  • 인텔® 이더넷 어댑터 전체 드라이버 팩. 이 옵션에는 모든 운영 체제용 드라이버와 도구가 포함되어 있지만 다운로드 용량이 매우 큽니다.
참고:
  • 드라이버를 설치하려면 운영 체제에 대한 관리자 권한이 필요합니다.
  • 현재 릴리스에서 지원되는 Windows 버전에 대해서는 어댑터 사용자 가이드의 "지원되는 운영 체제" 페이지를 참조하십시오.
  • 인텔 PROSet이 모든 Windows 버전에서 지원되는 것은 아닙니다. 자세한 내용은 어댑터 사용자 가이드를 참조하십시오.

Windows 드라이버와 도구를 설치하려면 다음과 같이 하십시오.

  1. http://www.intel.co.kr/support에서 원하는 소프트웨어 패키지를 다운로드하십시오.
    • 기본 드라이버만 다운로드하는 경우:
      • 32비트 OS를 사용하고 있다면 Wired_driver_XX.X_32.zip을 다운로드하십시오.
      • 64비트 OS를 사용하고 있다면 Wired_driver_XX.X_x64.zip을 다운로드하십시오.
    • 인텔 PROSet만 다운로드하는 경우:
      • 32비트 OS를 사용하고 있다면 Wired_PROSet_XX.X_32.zip을 다운로드하십시오.
      • 64비트 OS를 사용하고 있다면 Wired_PROSet_XX.X_x64.zip을 다운로드하십시오.
    • 전체 드라이버 팩을 다운로드하는 경우:
      • Release_XX.X.zip을 다운로드하십시오.
  2. 다운로드한 소프트웨어 패키지를 하드 드라이브에 압축을 풉니다.
  3. 새 하드웨어 발견 마법사가 시작되면 취소 단추를 누릅니다.
  4. 전체 드라이버 팩을 다운로드한 경우:
    1. 압축을 푼 파일에서 \APPS\SETUP\SETUPBD로 이동한 다음, 사용 중인 Windows 버전(32비트 또는 64비트)에 해당하는 Windows 하위 폴더로 이동합니다.
    2. Winx64 또는 Win32 폴더 내에서 SetupBD.exe를 더블클릭합니다.
    3. 설치 마법사의 단계를 완료합니다.
    4. 인텔 PROSet을 설치하려면 \APPS\PROSETDX로 이동한 다음, 사용 중인 Windows 버전(32비트 또는 64비트)에 해당하는 Windows 하위 폴더로 이동합니다.
    5. Winx64 또는 Win32 폴더 내에서 DxSetup.exe를 더블클릭합니다.
    6. 설치 마법사의 단계를 완료합니다.
  5. Windows 드라이버만 다운로드한 경우:
    1. 다운로드한 파일의 압축을 푼 후, SetupBD.exe를 더블클릭합니다.
    2. 설치 마법사의 단계를 완료합니다.
  6. 인텔 PROSet을 설치하려는 경우:
    1. 아직 다운로드하지 않았다면, Wired_PROSet_XX.X_xx.zip에서 파일을 다운로드하고 압축을 풉니다.
    2. 압축을 푼 파일에서 DxSetup.exe를 더블 클릭합니다.
    3. 설치 마법사의 단계를 완료합니다.

Linux*

이 드라이버의 바이너리 RPM* 패키지를 빌드하려면 'rpmbuild -tb <filename.tar.gz>'를 실행합니다. 여기서 파일 이름 <filename.tar.gz>는 드라이버의 해당 파일 이름으로 바꾸십시오.

참고:
  • 빌드가 올바르게 작동하려면 현재 실행 중인 커널과 설치된 커널 소스의 버전 및 구성이 일치해야 합니다. 커널을 다시 컴파일한 경우에는 시스템을 다시 부팅합니다.
  • RPM 기능은 Red Hat 버전에서만 테스트되었습니다.
  1. 선택한 디렉토리로 기준 드라이버 tar 파일을 이동합니다. 예를 들어, '/home/username/ixgbe' 또는 '/usr/local/src/ixgbe'를 사용할 수 있습니다.
  2. 파일의 압축을 풉니다. 여기서 <x.x.x>는 드라이버 tar 파일의 버전 번호입니다.
    tar zxf ixgbe-<x.x.x>.tar.gz
  3. 드라이버 src 디렉토리로 이동합니다. 여기서 <x.x.x>는 드라이버 tar 파일의 버전 번호입니다.
    cd ixgbe-<x.x.x>/src/
  4. 드라이버 모듈을 컴파일합니다.
    make install

    바이너리가 다음과 같이 설치됩니다.

    /lib/modules/<커널_버전>/kernel/drivers/net/ixgbe.o

    위에 보여진 설치 위치가 기본 위치입니다. 일부 Linux 배포판에서는 이러한 위치가 다를 수도 있습니다.

  5. 모듈을 설치합니다.
    modprobe ixgbe <parameter>=<value>
  6. 다음 명령을 실행하여 인터페이스의 IP 주소를 지정합니다. 여기서 <x>는 인터페이스 번호입니다.
    ifconfig eth<x> <IP_address>
  7. 인터페이스가 작동하는지 확인합니다. 다음 명령을 실행합니다. 여기서 <IP_주소>는 테스트할 인터페이스와 같은 서브넷에 있는 다른 컴퓨터의 IP 주소입니다.
    ping <IP_address>

FreeBSD*

이 드라이버는 FreeBSD 커널, 버전 4.8 이상에서 사용하기 위해 개발된 것입니다.

참고: 드라이버 모듈을 컴파일하려면 커널 소스가 설치되어 있어야 합니다.

아래 설명에서 x.x.x는 드라이버 tar 파일의 이름에 표시되어 있는 드라이버 버전입니다.

  1. 선택한 디렉토리로 기준 드라이버 tar 파일을 이동합니다. 예를 들어, /home/username/ixgb 또는 /usr/local/src/ixgb를 사용합니다.
  2. 파일 압축 풀기:
    tar xfz ixgb-x.x.x.tar.gz

    ixgb-x.x.x 디렉토리가 생성됩니다.

  3. man 페이지 설치:
    cd ixgb-x.x.x
    gzip -c ixgb.4 > /usr/share/man/man4/ixgb.4.gz
  4. 실행 중인 시스템에 드라이버를 로드하려면 다음 단계를 수행하십시오.
    cd ixgb-x.x.x
    make

    또는

    cd ixgb-x.x.x/src
    make load
  5. IP 주소를 인터페이스에 할당하려면 다음을 입력하십시오.
    ifconfig ixgb<interface_num> <IP_address>
  6. 인터페이스가 작동하는지 확인합니다. 다음 명령을 실행합니다. 여기서 <IP_주소>는 테스트 중인 인터페이스와 같은 서브넷에 있는 다른 컴퓨터의 IP 주소입니다.
    ping <IP_address>
  7. 시스템을 부팅할 때 드라이버를 자동으로 로드하려면
    cd ixgb-x.x.x/src
    make load
    cp if_ixgb.ko /modules

    /boot/loader.conf를 편집하고 다음 행을 추가합니다.

    if_ixgb_load="YES"

    또는

    드라이버를 커널로 컴파일합니다(항목 8 참조). /etc/rc.conf를 편집하고 ifconfig_ixgb<interface_num> 항목을 생성합니다.

    ifconfig_ixgb<interface_num>="<ifconfig_settings>"

    사용 예제:

    ifconfig_ixgb0="inet 192.168.10.1 netmask 255.255.255.0"
  8. 커널에 드라이버를 컴파일하려면 다음을 입력하십시오.
    cd ixgb-x.x.x/src
    mkdir /usr/src/sys/dev/ixgb
    cp if_ixgb* /usr/src/sys/dev/ixgb
    cp ixgb* /usr/src/sys/dev/ixgb
    cp Makefile.kernel /usr/src/sys/modules/ixgb/Makefile

    /usr/src/sys/conf/files.i386 파일을 편집하고 다음 행을 추가합니다.

    dev/ixgb/ixgb_hw.c optional ixgb
    dev/ixgb/ixgb_ee.c optional ixgb
    dev/ixgb/if_ixgb.c optional ixgb

    /usr/src/sys/conf/files.i386 파일에서 다음 행(있는 경우)을 제거합니다.

    /dev/ixgb/if_ixgb_fx_hw.c optional ixgb
    /dev/ixgb/if_ixgb_phy.c optional ixgb

    /usr/src/sys/i386/conf에서 커널 구성 파일(즉, GENERIC 또는 MYKERNEL)을 편집하고 다음 행이 있는지 확인합니다.

    device ixgb

    커널을 컴파일하고 설치합니다. 커널 업데이트를 적용하려면 시스템을 재부팅해야 합니다.

커널 컴파일에 대한 자세한 내용은 FreeBSD 운영 체제 문서를 참조하십시오.
